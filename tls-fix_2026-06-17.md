# TLS 连接超时修复 — 2026-06-17

## 问题

`POST /api/generate/image` 返回 500 错误：`fetch failed`。Node.js 连接 `ark.cn-beijing.volces.com:443` 超时。

## 根因

Node.js 22 (OpenSSL 3.0.16) 与火山引擎 Ark API 服务器的 TLS 握手不兼容：
- Node.js 默认尝试 TLS 1.3 协商 → 服务器无响应 → socket hang up
- 显式指定 `--tls-min-v1.2` 降级为 TLS 1.2 → 连接成功（返回 401，预期行为）

验证链路：
1. `Test-NetConnection ark.cn-beijing.volces.com -Port 443` → TCP 连通 ✅
2. `curl.exe https://ark.cn-beijing.volces.com/` → HTTP 401 ✅
3. `node -e "https.get('https://ark.cn-beijing.volces.com')"` → 超时 ❌
4. `node --tls-min-v1.2 -e "https.get('https://ark.cn-beijing.volces.com')"` → HTTP 401 ✅

## 修复

`package.json` 的 dev/start 脚本添加 `NODE_OPTIONS=--tls-min-v1.2`：

```json
"dev": "set NODE_OPTIONS=--tls-min-v1.2 && next dev",
"start": "set NODE_OPTIONS=--tls-min-v1.2 && next start"
```
