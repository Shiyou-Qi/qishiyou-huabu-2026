## MentionInput 提示词无法输入修复

### 问题
- AI生图节点的 MentionInput 提示词输入框点不进去、无法输入文字

### 根因
- `components/ui/mention-input.tsx` 的 `buildEditor` 在空状态下插入 `<br><span class="mention-ph">placeholder</span>` 作为"空 div 占位 + 提示"
- contenteditable `<div>` 里**只有 `<br>` 没有可聚焦的文本节点**，浏览器把光标定位在 `<br>` 之前失败 —— 用户点击输入框时**光标根本没进去**

### 修复
1. `components/ui/mention-input.tsx`
   - `buildEditor` 空状态从 `<br>+<span class="mention-ph">` 改为 `textContent = '\u200B'`（zero-width space）—— 让 div 有可聚焦的文本节点，光标能进入
   - 标记 `el.dataset.empty = 'true' | 'false'`，由 CSS 决定是否显示 placeholder
   - `flush` 同步维护 `data-empty`；清空输入时塞回 ZWS，匹配 useEffect 重 build 时的占位
2. `app/globals.css` 新增 `[contenteditable][data-empty="true"]::before { content: attr(data-placeholder); }` —— placeholder 文字改用 CSS 伪元素展示

## 提示词连接修复（video-node 提示词端口断连/重连对账）

   - `promptNodeId` 改为**仅依赖 edges**：`edges.find(e => e.target === id && e.targetHandle === PROMPT_HANDLE)?.source`
   - 删除 `data.promptNodeId` 的写入（mount 时也不再写）
   - 删除 `useMemo` 里对 `data` 的依赖

- top% 是相对**整个 wrapper 高度**算的（不是相对 body），所以**卡片总高变化时**（options 折行、底栏换行等）端口位置会有偏差
- 当前 4 个值是按"估算 280px 卡片高"算的近似值，**如果实际跑起来偏差大再调**
- 终极稳定方案是用 `ResizeObserver` 测 wrapper 实际高度后用 JS 动态设 top，但当前不必要

## video 节点端口位置：硬编码 top% → DOM 动态测量（用 ResizeObserver 实时跟 tab 按钮）

### 用户反馈
- 端口位置和 4 个 tab 按钮中心完全对不上
- 截图显示：tab"文生视频"在 25% 高度，但端口"文生视频"却在 35% 高度

### 根因
- 之前方案用硬编码 top%（'12%' '24%' '35%' '45%' '55%'）
- 但 wrapper 总高度依赖**卡片实际内容高度**（options 折行、生成中预览变化、底栏多行等都会变）
- 任何**硬编码值**都只在特定卡片高度下精准，**实际一定有偏差**

### 修复（最终稳定方案）
1. `components/nodes/node-base.tsx`
   - `NodeBase` 改成 `forwardRef<HTMLDivElement, NodeBaseProps>`，把最外层 div ref 透出
2. `components/nodes/video-node.tsx`
   - `wrapperRef = useRef<HTMLDivElement>(null)` 拿 NodeBase 透出的 wrapper
   - `tabBtnRefs = useRef<Partial<Record<Tab, HTMLButtonElement>>>({})` 拿 4 个 tab 按钮 DOM
   - `portTops: Record<string, string>` state 存动态算出的 top%
   - `useLayoutEffect` 测量：
     - wrapper `getBoundingClientRect()` 拿高度
     - 每个 tab 按钮相对 wrapper 顶部的 `centerY`，除以 wrapper 高度得百分比
     - "提示词"端口 = 第 1 个 tab 按钮上方 1/2 按钮高度
   - `ResizeObserver(wrap)` 监听 wrapper 尺寸变化（窗口缩放 / options 折行 / 生成预览变化），自动重新测量
   - `inputPorts` 全部传 `top: portTops[key] ?? '50%'`（未测量前用 50% 占位）
   - tab 按钮加 `ref={(el) => { tabBtnRefs.current[t.id] = el }}` 收集 DOM
